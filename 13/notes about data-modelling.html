Data Modelling

Important Notes

  • NO DATA REPLICATION
  • Split the replicated rows in to multiple tables
  • Make INT Ids for every table that you can reference
  • When you find many different rows that correlate with the same row, then it's probably a one-to-many relation

How to represent a one-to-many-relationship

  1. Make an primary key column (mostly called "id") for each table
  2. To reference a table (the one) in another table (the many) we use a foreign key column that contains the table_ids of the other table
